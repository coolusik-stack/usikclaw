# OpenClaw Notes — Supanova Design Skill

이 레포는 앞으로 웹페이지형 보고서/리포트 디자인 참고용으로 사용한다.

## 우선 적용 원칙
- standalone HTML 우선
- 한국어 타이포그래피 품질 유지
- 정보 밀도보다 가독성과 프리미엄 레이아웃 우선
- 카드/섹션 구조, 여백, 시각적 위계에 특히 신경쓰기
- 단순 문서가 아니라 '바로 보여줄 수 있는 결과물' 느낌으로 제작

## 보고서에 특히 가져올 것
- 강한 헤더/히어로 섹션
- 카드형 정보 구조
- 명확한 섹션 위계
- 어두운 배경을 쓰더라도 PDF 인쇄 대응 스타일 별도 고려
- 모바일에서도 읽히는 반응형 구성

## 실무 적용 메모
- 랜딩페이지용 요소는 그대로 복제하지 말고, 리포트/브리핑 형식에 맞게 번안할 것
- 보고서 생성 시 README/HTML/PDF 세트로 마무리
- PDF 변환 시 print stylesheet를 별도로 챙길 것
