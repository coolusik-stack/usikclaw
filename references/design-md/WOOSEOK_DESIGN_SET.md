# Wooseok Design Set

우석 작업에 바로 쓰기 좋은 DESIGN.md 레퍼런스 5종.

## 포함된 레퍼런스
- `claude/`
- `linear/`
- `vercel/`
- `raycast/`
- `supabase/`

각 폴더에는 아래 파일이 있음:
- `DESIGN.md` — 에이전트가 읽기 좋은 디자인 시스템 설명
- `preview.html` / `preview-dark.html` — 시각 참고용
- `README.md`

---

## 추천 사용처

### 1. Claude
**추천 작업**
- 아침 브리핑
- 리서치 페이지
- 정보형 콘텐츠 페이지

**왜 맞는가**
- 차분함
- 에디토리얼 톤
- AI가 만든 티가 상대적으로 덜 남

### 2. Linear
**추천 작업**
- 대시보드
- CRM형 페이지
- 운영용 실무 UI

**왜 맞는가**
- 가장 정돈된 제품 UI 감각
- 정보 밀도 조절이 좋음
- 실무 화면에 강함

### 3. Vercel
**추천 작업**
- BOOL 브랜드 소개 페이지
- 제품/회사 소개 랜딩
- 비즈니스용 소개 사이트

**왜 맞는가**
- 단단한 블랙 기반 톤
- 기술 브랜드 감각
- 신뢰감이 큼

### 4. Raycast
**추천 작업**
- 브리핑 페이지 리디자인
- 툴/자동화 소개 페이지
- 앱 같은 인터페이스

**왜 맞는가**
- 다크 테마가 세련됨
- UI가 polished
- 너무 차갑지 않음

### 5. Supabase
**추천 작업**
- 데이터 많은 대시보드
- BOOL 운영 페이지
- 기능 설명형 랜딩

**왜 맞는가**
- 구조적
- 기술적 신뢰감
- 다크+포인트 컬러 균형이 좋음

## 추천 조합

### 브리핑 페이지
- `claude` + `raycast`

### BOOL 아웃리치 대시보드
- `linear` + `supabase`

### BOOL 브랜드/소개 페이지
- `vercel` + `claude`

---

## 사용 방법

### 방법 1 — 단일 레퍼런스 적용
작업 전에 해당 `DESIGN.md`를 읽고,
"이 스타일 원칙을 유지해서 페이지를 만든다"고 명시.

### 방법 2 — 하이브리드 적용
예:
- 구조는 `linear`
- 톤은 `claude`
- 다크 감도는 `raycast`

### 방법 3 — 미세조정
사용자가 피드백 주면 다음처럼 조정:
- `Linear보다 덜 차갑게`
- `Vercel보다 정보 밀도 높게`
- `Claude 톤이지만 더 고딕하게`

## claw 권장
우석 작업 기준 첫 적용 우선순위:
1. 브리핑 페이지 → `claude` / `raycast`
2. 영업 대시보드 → `linear`
3. BOOL 브랜드 페이지 → `vercel`
