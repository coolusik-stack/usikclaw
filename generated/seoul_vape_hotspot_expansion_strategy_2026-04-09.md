## 0. 기준점
- 실데이터 CSV 기준 현재 행 수: **318개**
- 그중 `010 / 0507` 문자 가능 번호: **279개**
- 기타 번호(02 / 070 / 080 등): **39개**
- 참고: `generated/vape_500_progress.md`의 284개와 차이가 있어, **운영 상태 파일과 CSV 총량을 먼저 한 번 맞춰두는 것**이 좋다.

핵심 해석은 간단하다.
1. 이미 서울 전역 기본 골격은 잡혔다.
2. 이제는 무작정 넓히기보다, **핫스팟 빈칸 메우기 + 0507/010 비율 높은 존 집중 + 중복/번호 업그레이드**가 효율적이다.
3. 특히 현재 수량이 낮은데 상권 밀도는 높은 구간, 예를 들면 **삼성/코엑스, 여의도, 마포/공덕, 이태원, 목동/오목교, 건대/왕십리**가 다음 확장 포인트다.

## 1. 우선순위 높은 서울 핫스팟 시드 키워드

아래는 실제 확장 효율 기준의 우선순위다. 앞쪽일수록 먼저 긁는 게 좋다.

### Tier A, 바로 다음 배치에 넣을 지역
1. **강남역 / 신논현 / 역삼 / 선릉 / 삼성 / 코엑스**
   - 이유: 밀집도 높고 브랜드 멀티샵, 오피스 상권, 지하상가형 매장까지 같이 잡힘
2. **홍대 / 합정 / 상수 / 연남 / 신촌 / 이대**
   - 이유: 이미 검증된 상권이고, 변형 상호와 서브매장이 계속 나온다
3. **성수 / 서울숲 / 뚝섬**
   - 이유: 현재 수량 대비 상권 잠재력이 큼, 최근 운영 흔적 있는 매장 비율이 높다
4. **잠실 / 송리단길 / 석촌 / 송파 / 문정 / 가락**

## 3. 존별 수집 전략

### Zone 1. 강남 비즈니스 코어
포함: 강남역, 신논현, 논현, 역삼, 선릉, 삼성, 코엑스
- 1차: 역명 키워드로 지하상가/오피스 빌딩형 매장 확보
- 2차: 발견된 매장명을 다시 검색해 0507 전환 여부 확인
- 3차: 같은 매장의 브랜드명/약칭 변형까지 재검색
- 기대값: **다중 출처로 같은 매장이 반복 노출**되므로 QC만 잘하면 고품질 누적이 쉬움

### Zone 2. 홍대-연남-신촌 대학가 코어
포함: 홍대, 합정, 상수, 연남, 신촌, 이대, 아현
- 1차: 역명 + 동명 둘 다 검색
- 2차: 인스타/블로그 노출 매장 위주로 최근 운영 흔적 체크
- 3차: 연남/서교/창천/대현처럼 행정동 변형도 검색
- 기대값: 리뷰 많은 상점, 상호 변형, 지하/골목 상점이 잘 나옴

### Zone 3. 성수-건대-왕십리 연결축
포함: 성수, 서울숲, 뚝섬, 건대입구, 군자, 자양, 왕십리, 한양대
- 1차: 성수는 `성수동`, `성수역`, `서울숲`을 따로 돈다
- 2차: 건대권은 `건대`, `건대입구`, `군자`, `중곡`, `자양`을 묶어서 확장
- 3차: 같은 전화번호가 2개 지점에 걸쳐 보이면 지점 분리 여부 확인
- 기대값: 현재 보유량이 아직 상권 잠재력보다 적어서 추가 수확 여지 큼

### Zone 4. 잠실-송파-문정 생활소비축
포함: 잠실, 송리단길, 석촌, 송파, 문정, 가락, 장지
- 1차: 역세권 검색
- 2차: 주거상권 키워드 검색
- 3차: `OO전자담배 잠실점`, `OO 송파점` 식 지점형 검색
- 기대값: 0507 확보율이 좋고, 운영 중 흔적이 비교적 선명함

### Zone 5. 영등포-여의도 오피스 혼합축
포함: 영등포, 문래, 당산, 여의도, 국회의사당, 샛강
- 1차: 역명 검색으로 오피스텔/상가 지하 매장 찾기
- 2차: 이미 확보한 `no_place_id` 후보를 매장명 단독 검색으로 재검증
- 3차: 동일 주소 내 호수 차이, 지하층 차이 체크
- 기대값: `문래/당산/영등포`에서 추가 매장이 계속 나올 확률이 높음

### Zone 6. 생활상권 보강축
포함: 사당, 방배, 교대, 양재, 목동, 오목교, 노원, 공릉, 서울대입구, 신림, 마곡
- 1차: 역세권 검색
- 2차: 동명 검색
- 3차: 브랜드 취급점/스토어 로케이터 대조
- 기대값: 밀집도는 낮지만 중복 적고 신규 확보율이 괜찮음

## 4. 중복 제거 및 품질관리 규칙

### 4-1. 중복 판정 기본키
다음 순서로 본다.
1. **전화번호 동일**면 최우선 중복 후보
2. 전화가 다르더라도 **주소 + 상호 핵심어**가 같으면 같은 매장 가능성 높음
3. 같은 브랜드의 여러 지점은 **지점명 또는 주소가 다르면 분리 유지**
4. 상호가 달라도 `네이버 플레이스 링크 / 지도 링크 / 동일 건물 동일 호수`면 재확인

### 4-2. 상호 정규화 규칙
저장 전 상호에서 아래 요소는 비교용 보조 필드로 정규화한다.
- 공백 제거
- `전자담배`, `베이프`, `VAPE`, `멀티샵`, `점`, `본점` 같은 반복어는 비교 시 약하게 본다
- 괄호, 특수문자, 영문 대소문자 차이는 무시
- 예: `고릴라파이프(연남동점)` = `고릴라파이프 연남동점`

### 4-3. 주소 QC
- 서울 외 주소는 제외
- 단, 현재 큐를 보면 `status: non_seoul`인데 실제 verified 내용은 서울인 케이스가 있다
- 따라서 **상태 라벨보다 실제 verified address를 우선**한다
- 지번과 도로명은 서로 변환 가능하므로 둘 중 하나만 맞아도 후보 유지

### 4-4. 번호 QC
- 1순위 반영: `010`, `0507`
- 2순위 보관: `02`, `070`, `080`
- 같은 매장의 `02`와 `0507`이 같이 나오면 **0507로 업그레이드 저장**, 기존 번호는 notes에 남김
- 번호가 마스킹되었거나 고객센터 성격이면 `재확인 필요`

### 4-5. 운영 흔적 QC
아래 2개 이상이면 유효 후보로 본다.
- 네이버 플레이스 리뷰 존재
- 최근 블로그/인스타 업로드 흔적
- 영업시간 노출
- 길찾기/저장/전화 버튼 정상 노출

### 4-6. 보류 규칙
다음은 바로 원장에 넣지 않는다.
- 주소가 서울인지 불명확
- 번호가 없거나 마스킹만 존재
- 브랜드 로케이터에는 있는데 실제 플레이스 확인이 안 됨
- 이미 있는 매장과 전화/주소 충돌이 해결 안 됨

## 7. 추천 실행 순서

### 가장 추천하는 실제 순서
1. **운영 상태 파일 숫자와 CSV 총량 동기화**
2. **삼성/코엑스, 여의도, 마포/공덕, 이태원, 건대/왕십리**처럼 수량이 낮은 고가치 지역부터 메우기
3. `no_place_id` 큐를 매장명 단독 검색으로 한 번 더 털기
4. 원장 내 `02 / 070 / 080` 업그레이드 배치 따로 실행
5. 20개 단위 스프린트로 계속 누적

핵심은 이거다.
- **새 지역 개척보다, 이미 잘 먹히는 핫스팟을 더 촘촘하게 재탐색하는 편이 수율이 좋다.**
- 그리고 지금 단계에서는 **신규 발견 수**와 **문자 가능 번호 업그레이드 수**를 같이 KPI로 봐야 한다.
